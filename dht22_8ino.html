byte *  message,

)

Callback function for MQTT subscriptions.

This function is called whenever a message is received on a subscribed MQTT topic. It first deserializes the received message into a DynamicJsonDocument. Then, based on the topic of the received message, it controls two different GPIOs. If the topic is 'bed/temp', it controls the GPIO linked to a temperature lamp (templamp). If the topic is 'bed/hum', it controls the GPIO linked to a humidity lamp (humlamp). The lamps are turned ON or OFF based on the content of the received MQTT message.

Parameters
topicThe topic on which the message was received
messageThe received message
lengthThe length of the received message

void loop ( )

Main execution loop of the program.

This function is executed repeatedly by the ESP8266 microcontroller. It ensures that the device remains connected to the MQTT broker, and handles the publication of sensor data.

The loop reads data from the DHT22 sensor and publishes the humidity and temperature values to the MQTT broker every 2 seconds. If the sensor readings fail, it logs an error message and attempts to read again in the next iteration of the loop.

void reconnect ( )

Function to reconnect to MQTT broker.

This function loops until the ESP8266 is reconnected to the MQTT broker. If the connection attempt is successful, the function also re-subscribes to the topics "bed/temp" and "bed/hum".

Note: If there are issues with multiple MQTT connections, consider changing the ESP device ID which is given as the first argument to client.connect(). For example, you could change "ESP8266Client" to "ESP1_Office" for one device and to "ESP2_Garage" for another device to differentiate between them.

This function will wait for 5 seconds before retrying if a connection attempt fails.

void setup ( )

Function to setup ESP8266.

This function sets the ESP GPIOs to OUTPUT mode for controlling LEDs. It also starts the serial communication at a baud rate of 115200, sets up the WiFi connection, specifies the MQTT broker, and defines the callback function.

The callback function is the function that will be called when a subscribed topic receives a message. It is responsible for processing the received message and performing appropriate actions, such as controlling the LEDs in this example.
